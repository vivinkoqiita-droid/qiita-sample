# FileSystemObserver

`FileSystemWatcher` の基本を WinForms で確認するための学習用アプリです。  
監視先フォルダ、フィルタ、サブフォルダ対象、更新集約の有無を切り替えながら、作成・更新・削除・リネームの通知がどう出るかを一覧で見られます。

## このアプリで見られること

- `FileSystemWatcher` の基本的な使い方
- `Created` / `Changed` / `Deleted` / `Renamed` / `Error` の受け取り方
- `Changed` が複数回届く場面
- サブフォルダ監視の有無による違い
- 共有フォルダ監視で見直したい点
- WinForms での開始・終了位置
- UI スレッドへ戻して画面更新する流れ

## 使い方

### 1. 監視先を決める

- 起動直後はデスクトップが入る
- 別フォルダを見たいときは入力欄へパスを入れる
- フォルダをフォーム全体または入力欄へドラッグ&ドロップしても反映される

### 2. 監視条件を決める

- `フィルタ`
  - `*.*` : 全ファイル
  - `*.csv` : CSV のみ
  - `*.txt` : テキストのみ
- `サブフォルダ含む`
  - ON : 配下フォルダも監視対象
  - OFF : 直下のみ監視対象
- `更新をまとめて表示`
  - `Changed` が短時間に連続したときの見やすさ調整

### 3. 監視を始める

- `監視開始` を押す
- 開始後は監視条件の一部が編集不可になる
- 画面上部の状態表示で監視中かどうかを確認する

### 4. フォルダ内で操作する

たとえば次を試すと動きが分かりやすくなります。

- 新しいファイルを作る
- 既存ファイルを上書き保存する
- ファイル名を変える
- ファイルを削除する
- サブフォルダを作ってその中へファイルを置く

### 5. 一覧を見る

一覧には発生したイベントが追加されます。  
まずは `作成`、`更新`、`削除`、`リネーム` がどう並ぶかを見るだけで十分です。

### 6. 監視を終える

- `監視終了` を押す
- `一覧消去` で表示だけ消せる
- `フォルダ表示` で監視先をエクスプローラーで開ける

## 見かた

### まず見る場所

最初は次の 3 点を見ると流れをつかみやすくなります。

1. **どの操作でどのイベントが出るか**  
   作成・更新・削除・リネームの対応を見る
2. **`Changed` が何回出るか**  
   保存方法によって 1 回とは限らない点を見る
3. **対象範囲がどこまでか**  
   サブフォルダ ON/OFF で差を見る

### 一覧の読み方

- `作成`  
  新しいファイルやフォルダが作られたとき
- `更新`  
  内容更新、保存、コピー途中などで複数回出ることがある
- `削除`  
  ファイルやフォルダが消えたとき
- `リネーム`  
  名前変更時。旧パスと新パスの対応を見る
- `エラー`  
  監視異常。共有フォルダや通知集中時に特に確認したい

### 学習時の見どころ

- `*.*` と `*.csv` で通知量がどう変わるか
- サブフォルダ ON/OFF でどこまで拾うか
- 同じ保存操作でも `Changed` の回数が変わること
- コピー直後は相手側の書き込みが終わっていない場合があること
- 共有フォルダではローカルより乱れやすいこと

## 画面で触る項目

- **監視先**  
  監視対象フォルダ。起動直後はデスクトップを初期表示。
- **フィルタ**  
  `*.*` や `*.csv` のように監視対象を絞る入力欄。
- **サブフォルダ含む**  
  `IncludeSubdirectories` の切り替え。
- **更新をまとめて表示**  
  `Changed` が短時間に連続したときの見やすさ調整。
- **監視開始 / 監視終了**  
  監視インスタンスの生成、停止、解放の確認。
- **一覧消去**  
  表示リセット。
- **フォルダ表示**  
  監視先をエクスプローラーで開く確認用。
- **ドラッグ&ドロップ**  
  フォルダをフォーム全体か入力欄へ落として監視先へ反映。

## 学習の見どころ

### 1. 初期値と実行時切り替えの分け方

このアプリでは、次のように場所を分けています。

- `MainForm.Designer.cs`  
  `Text`、`Checked`、`AllowDrop`、`Click`、`DragEnter`、`DragDrop` など、起動直後に決まっている項目
- `MainForm.cs`  
  デスクトップ初期表示のような実行時に決まる値、監視開始後に変わる状態、監視ロジック本体

見た目の初期値と、監視中かどうかで変わる状態を分けておくと追いやすくなります。

### 2. WinForms での開始と終了

- 開始: `StartMonitoring()`
- 終了: `StopMonitoring()`
- フォーム終了時: `OnFormClosing()` から停止

開始時にイベント関連付けと `EnableRaisingEvents = true` を行い、終了時に `Dispose()` まで行う流れを見られます。

### 3. `Changed` の複数通知

保存の仕方やコピーの流れによって、`Changed` は 1 回とは限りません。  
このアプリでは一覧に残して、通知の出方を実際に確認できます。

### 4. UI スレッドへの戻し

監視イベントは UI スレッド以外から届くことがあります。  
そのため、一覧追加などの画面更新は `BeginInvoke` を通す構成にしています。

## ファイル構成

- `MainForm.cs`  
  監視開始・終了、イベント受信、一覧追加、状態表示切り替え
- `MainForm.Designer.cs`  
  画面部品、初期表示、イベント関連付け
- `FileEventEntry.cs`  
  一覧表示用データ
- `Program.cs`  
  起動処理

## まず試す流れ

1. アプリを起動
2. デスクトップか任意フォルダを監視先に指定
3. `*.*` のまま監視開始
4. 監視先で新規作成、上書き保存、削除、名前変更を試す
5. 一覧へどの順で通知が出るかを見る
6. 次に `*.csv` へ変えて通知量の差を見る
7. サブフォルダ含むを切り替えて範囲の差を見る
8. 共有フォルダでも同じ操作を試して違いを見る

## 詰まりやすい点

- `Changed` が複数回来る
- 受信直後は相手側の書き込み中で開けないことがある
- 共有フォルダでは権限や切断影響を受けやすい
- 監視範囲を広げすぎると通知が多くなる
- `InternalBufferSize` を増やす前に `Filter` と `NotifyFilter` を見直した方が早いことが多い
